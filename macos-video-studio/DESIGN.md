# macOS AI 视频工作台 — 技术设计文档

> 一个「会自己搭流程」的 macOS AI 视频制作工作台：用户用自然语言描述意图，Agent 负责把繁琐的
> ComfyUI 工作流搭好、调好参数、跑起来，并全程把进度 / 中间产物 / 预览喂回给用户；用户随时可切到
> 无限画布手动微调。面向「AI 视频入门」与「生产力工具」双场景。

状态：设计基线（v0.1，2026-06）。本文是后续 PoC / 实现的依据。

---

## 1. 定位与核心理念

- **一句话**：把 AI 叙事视频里最痛、最繁琐的「资产一致性 + 分镜 + 图生视频 + 选片」这条链路，
  交给 Agent 编排，用户用对话驱动、用画布微调。
- **核心理念：图 / 项目是唯一事实来源（single source of truth）。** Agent 和人都只是同一份
  数据的编辑者；执行引擎（本地 ComfyUI / 云）都被抽象成「能消费这份数据的后端」。这样
  「Agent 搭 ↔ 人手调」天然互通、可互相接管。

### 双场景、单引擎、两种外壳（可切换）

不做两个产品，而是**同一套引擎 + 两种布局模式**：

- **对话优先**（入门）：默认隐藏节点图，主打「对话 + 预览 + 少量旋钮」。画布是可选高级视图。
- **画布优先**（生产力）：画布为主视图，强调可复现 / 版本化 / 批量 / 精细控参，Agent 作为加速器。

实现上是 `NavigationSplitView` 的布局切换，底层共享同一数据模型与事件流。

## 2. 范围

### MVP（本期目标）—— 「项目主链路」垂直切片

> 描述角色 → 出多视角定稿 → 锁成角色档案 → 给一个场景生成 2~3 个分镜关键帧 →
> 对选中关键帧图生视频出几条 take → 选片 → **导出工程 / 片段**

覆盖阶段：**前期（拆镜头）→ 资产 → 分镜 → 生成 → 选片 → 导出**。

### 非目标（后置 / 后续独立 Agent）

- **剪辑 / 后期**：剪辑、调色、字幕、和片。后续由**独立的「剪辑 Agent」**单独做。
- **口型同步 / 对白配音**：与剪辑批次一起做（片型是有剧情的角色片，最终需要）。
- 内置 NLE 时间线（MVP 走「导出到剪映 / DaVinci」收尾，即下文 B 方案）。

## 3. 总体架构（client / server）

由于 RTX 5090 在**本地千兆局域网的独立 Linux 机器**上，架构是「Mac 客户端 + 5090 渲染/编排服务器」。
Mac 只负责 UI、画布与预览；重活全在 5090 机器上。

```
   ┌──────────────── Mac (SwiftUI 客户端) ────────────────┐
   │  可切换布局: 对话优先 ⇄ 画布优先 (同一数据模型)        │
   │  ┌───────────┐   ┌──────────────────────────────┐   │
   │  │ Chat/Agent│   │ 自研无限画布 (SwiftUI+Metal) │   │
   │  │ 侧栏      │   │ 项目层 ⇄ 工作流层 双层视图     │   │
   │  └─────┬─────┘   └───────────────┬──────────────┘   │
   │        └──────── Project / Graph IR (唯一事实来源) ──┘ │
   └───────────────────────┬──────────────────────────────┘
                            │  HTTP/WS (千兆局域网直连)
   ┌────────────── 5090 机器 (Linux, headless) ────────────┐
   │  ┌──────────────────────────┐                         │
   │  │ Orchestrator (Python/     │  ← Agent 运行时在这     │
   │  │ FastAPI)                  │     (靠近 schema / 资产)│
   │  │ - 导演 Agent + 搭图 Agent │                         │
   │  │ - 配方库 + /object_info   │                         │
   │  │   schema 索引(RAG) + 校验 │                         │
   │  │ - 任务队列 / 成本估算 / 路由│                        │
   │  │ - 项目存储 / 资产库        │                         │
   │  └───────┬───────────┬──────┘                         │
   │   ┌──────▼─────┐  ┌──▼──────────────────────────┐    │
   │   │ ComfyUI    │  │ 云适配器                      │    │
   │   │ (本地5090) │  │ 即梦(火山引擎) / Qwen(DashScope)│  │
   │   └────────────┘  └─────────────────────────────┘    │
   └───────────────────────────────────────────────────────┘
```

**为什么 Agent / 编排放在 5090 机器上**：改图前要对照 ComfyUI 的 `/object_info` 真实 schema 做校验、
要跑 RAG 检索配方、要管云任务轮询、要读写大体积素材——这些在 Python 生态里最省事，且离 ComfyUI 和
资产最近。Mac 端保持纯客户端，将来加 iPad / Web 客户端不用重写。

## 4. 核心数据模型

产品不是「一张 ComfyUI 图」，而是「一个项目」。一支片子横跨几十个生成任务、要管资产复用与一致性，
因此核心模型是项目结构：

```
Project（一支片子）
├─ meta: 标题 / 风格设定 / 宽高比 / 帧率 ...
├─ AssetLibrary / Character Bible        ← 一致性的锚点
│   ├─ Character[]   定稿图集 + 身份特征(PuLID/InstantID) + 可选 LoRA + 触发词
│   ├─ Wardrobe[]    服装
│   ├─ Prop[]        道具
│   ├─ Environment[] 场景背景设定图
│   └─ StyleFrame[]  美术风格 / lookdev
├─ Shots[]                                 ← 分镜
│   └─ Shot
│       ├─ 脚本: 谁 / 干什么 / 景别 / 运镜 / 时长 / 对白
│       ├─ refs: 用到的 Character / Wardrobe / Environment ...
│       ├─ keyframes: 起始帧 (±尾帧)
│       ├─ takes[]: 多条生成结果 (本地或云)
│       └─ selectedTake
│       └─ graph: 该镜头背后的 ComfyUI 工作流 (Graph IR)
└─ Timeline（后置，剪辑 Agent 阶段）
```

### Graph IR

与 ComfyUI 的 `prompt` / `workflow` JSON **一一映射**的中立数据结构（节点、inputs、links、widget 值）。
画布只是 IR 的可视编辑器；Agent 改的也是同一份 IR；导出时序列化成 ComfyUI API 格式。

```jsonc
// Graph IR 概要
{
  "nodes": [
    { "id": "n1", "type": "KSampler",
      "widgets": { "seed": 123, "steps": 25, "cfg": 7.0, "sampler_name": "euler" },
      "inputs":  { "model": {"node":"n0","slot":0}, "positive": {"node":"n2","slot":0} },
      "pos": [120, 80] }
  ],
  "links": [ /* 由 inputs 推导，或显式存 */ ]
}
```

## 5. 双层无限画布（SwiftUI 自研）

无限画布对应两个缩放层级，正好契合「项目」结构：

- **项目层画布**：节点 = 角色 / 资产 / 镜头；连线 = 「这个镜头用了这个角色 + 这个背景」。
  一眼看全片结构与资产复用关系。
- **工作流层画布**：双击某镜头钻进去，就是那张真正的 ComfyUI 节点图，手动微调。

### SwiftUI 实现要点

- **渲染分层**：节点用 SwiftUI 视图放进 `ZStack`，整体套一个视口变换（平移 / 缩放矩阵）；
  连线与网格用 `Canvas`（节点多了再换 Metal / CALayer）单独画。
- **手势**：`MagnificationGesture` + `DragGesture` 维护 `viewportTransform`。
- **预览**：图片 `AsyncImage`，视频 `AVKit (VideoPlayer)`，直接贴在产出节点上。
- **分阶段铺能力**（控制工作量）：
  1. MVP：**展示 + 轻编辑**——Agent 搭好图，用户改 widget 值、拖节点、连 / 断线、从节点库增删节点。
  2. 后续：框选、分组、复制子图、撤销重做栈、对齐吸附。

## 6. Agent 体系

分两层（剪辑 Agent 后置，是第三个）：

- **导演 / 制片 Agent**：拆剧本 → 镜头清单、排镜头、管一致性、调度本地 / 云、做高层规划。
- **搭图 Agent**：为每个具体生成任务造 / 改 ComfyUI 工作流（Graph IR）。
- **剪辑 Agent（后置）**：剪辑 / 调色 / 字幕 / 口型同步 / 和片。独立迭代。

### 可靠性策略（不让 LLM 凭空写整张图）

1. **配方库优先**：维护一批验证过的模板（文生图人物定稿、多视角、inpaint 合成关键帧、
   图生视频、首尾帧、补帧、超分……）。Agent 主要做「选配方 → 填参 → 局部改图」，而非发明拓扑。
2. **用 `/object_info` 做事实约束**：索引当前 ComfyUI 环境的真实节点 schema，改图前后都校验
   （节点是否存在、类型是否匹配、必填项是否齐），杜绝幻觉节点。
3. **图编辑原语当工具**（而非吐整段 JSON）：
   `search_recipes` / `instantiate_recipe` / `add_node` / `connect` / `set_param` /
   `delete_node` / `replace_subgraph` / `validate` / `estimate(cost,time)` /
   `run` / `run_to(node)`（局部执行看中间结果）。
4. **澄清式对话**：信息不足时主动问关键项（时长 / 分辨率 / 帧率、风格参考、本地还是云）。
5. **可解释**：每次改图附「为什么这么搭 / 这么调参」，便于用户学习与接管。

## 7. 一致性方案（核心竞争力）

角色 / 服装 / 风格要在所有镜头里保持一致——这是 AI 叙事视频成败关键，做成一等公民：

- 角色定稿后，把「定稿图集 + 身份 embedding（PuLID / InstantID）+ 可选用 5090 训的 LoRA + 触发词」
  打包成 **角色档案（Character Bible 条目）**。
- 之后所有镜头生成时，Agent **自动注入**这套档案（参考图 + IPAdapter / identity 锁定 + 触发词），
  用户不必每次手动设。
- 服装 / 道具 / 风格同理，可复用、可锁定。
- 这正是「Agent 替你管繁琐参数」最有价值的体现。

## 8. 制作管线与本地 / 云路由

完整管线（MVP 覆盖到「选片 / 导出」，剪辑及之后后置）：

**0. 前期**：创意 → 剧本 → 镜头清单（谁 / 干什么 / 景别 / 运镜 / 时长 / 对白）。Agent 主场。
**1. 资产**：角色多视角定稿 + 表情姿态表 / 服装 / 道具 / 场景背景 / 美术风格。
**2. 分镜**：每镜头起始帧构图（角色 + 服装 + 道具摆进背景）+ 动线 + 需要时出尾帧。
**3. 生成**：图生视频（±尾帧）+ 运镜 / 动作控制 + 每镜头多 take。
**4. 选片**：多 take 对比选优 + 补帧 / 超分。
**5. 导出**：导出片段 + 工程（给外部 NLE 收尾，B 方案）。
**(后置) 后期**：剪辑 / 音轨 / 口型 / 调色 / 字幕 / 和片 —— 独立剪辑 Agent。

### 本地 / 云分工（5090 本地为主，即梦 / Qwen 云补充）

| 阶段 | 主力 | 说明 |
|---|---|---|
| 定稿图 / 服装 / 道具 / 背景 / 风格 | **本地 5090** | Flux / Qwen-Image / SDXL + 一致性工具；可训 LoRA |
| 分镜关键帧合成 | **本地 5090** | inpaint / controlnet / 区域控制 |
| 图生视频镜头 | **本地 Wan 2.x ↔ 云即梦 / Qwen 路由** | 本地出草稿 / 可控；高质量或特定运镜走云。Agent 按镜头选 |
| 补帧 / 超分 | 本地 5090 | |
| 剪辑 / 调色 / 字幕 / 音轨 / 口型 | 后置（剪辑 Agent） | ComfyUI 不擅长，独立做 |

**智能路由**：Agent 按「质量 / 速度 / 成本 / 可用性」选后端；允许用户固定偏好
（如「草稿走本地、定稿走即梦」）。每次云调用**先预估、再确认、累计账单**。

## 9. 反馈系统

ComfyUI 后端通过 WebSocket 推 `progress`（采样步数）、`executing`（当前节点）、
`executed`（节点产物，含预览图）等事件。Orchestrator 聚合后转发给 Mac，画布做叠加层：

- **节点级进度**：执行中节点高亮 + 进度条；完成打勾；失败标红并把报错翻成人话。
- **实时中间预览**：采样过程用 TAESD / latent 预览「正在长出来的画面」；出图 / 出帧节点贴缩略图。
- **管线总进度 + 预计剩余 + 预计花费**（云模型尤其重要）。
- **产物时间线 / 版本**：每次运行产出留档，可对比、回滚、「基于这次再改」。
- **断点续跑 / 局部重跑**：改后半段参数只重算受影响下游（ComfyUI 缓存天然支持）。
- **选片视图**：多 take 并排对比、打分、Agent 可做自动初筛。

## 10. 技术栈

- **Mac 客户端**：SwiftUI（自研无限画布）+ Metal（连线 / 大图层）+ AVKit（视频预览）。
  云密钥存 **Keychain**。
- **Orchestrator**：Python + FastAPI，和 ComfyUI 同机（5090 Linux）。承载 Claude tool-use 循环、
  配方库、schema 索引（RAG）、任务队列、云适配器、项目 / 资产存储。
- **Agent 模型**：Claude（Fable 5 / Opus）+ tool use。
- **执行引擎**：ComfyUI（本地 5090，Linux headless）。
- **云**：即梦（火山引擎 API）、Qwen 通义万相（DashScope / 百炼）；异步作业模型
  `submit → task_id → poll → 下载`，适配器管轮询 / 重试 / 限流 / 密钥 / 费用。

---

## 11. client/server 工程要点

- **连接**：本地千兆局域网直连 Mac ↔ 5090；若需外网访问再加 Tailscale，避免裸公网暴露 ComfyUI。
- **资产传输**：后端为产物的存储 + 下载源，Mac 端做缓存；千兆 LAN 下大视频传输不是瓶颈。
- **断线恢复**：任务在服务器跑且状态持久化，Mac 重连后拉回进行中任务状态与已完成产物。
- **5090 环境**：Blackwell 需较新 CUDA / PyTorch；Linux headless 跑 ComfyUI。
- **API 契约（草案）**：
  - `POST /projects` / `GET /projects/{id}` — 项目 CRUD
  - `POST /chat` — 驱动 Agent（SSE 流式返回思考 / 工具调用 / 结果）
  - `POST /graphs/{id}/validate` / `POST /graphs/{id}/run` — 校验 / 执行
  - `POST /cloud/{provider}/submit` / `GET /cloud/tasks/{id}` — 云作业
  - `WS /events` — 进度 / 中间预览 / 产物事件
  - `GET /assets/{id}` — 资产下载

## 13. 里程碑

- **M1 — Orchestrator 骨架**：FastAPI + Claude tool-use + 接现有 ComfyUI；图编辑原语 + `validate`
  （对照 `/object_info`）+ `run`；跑通「自然语言 → 选配方 → 校验 → 出一张人物定稿图」。
- **M2 — 一致性**：角色档案（定稿图集 + identity + 触发词）+ 自动注入；多视角定稿配方。
- **M3 — 分镜 + 图生视频**：关键帧合成配方 + 图生视频配方（本地 Wan）+ 多 take。
- **M4 — SwiftUI 客户端**：对话侧栏 + 双层画布（展示 + 轻编辑）+ 可切换布局 + 远程连接配置。
- **M5 — 反馈链路**：远程 WS → 节点进度 / 中间预览 / 产物时间线 / 重跑 / 选片对比。
- **M6 — 云路由**：接入即梦或 Qwen，打通本地 ↔ 云路由 + 费用预估。
- **(后续) M7+ — 剪辑 Agent**：剪辑 / 音轨 / 口型 / 调色 / 字幕 / 和片。

## 14. 风险与未决问题

- **可靠性**：自动搭图出错率 —— 靠「配方 + schema 校验」压住。
- **一致性质量**：跨镜头角色 / 服装 / 风格漂移 —— 需在 M2 重点验证。
- **SwiftUI 自研画布工作量**：分阶段铺能力，MVP 只做展示 + 轻编辑。
- **云成本与限流**：必须预估 + 确认 + 配额保护。
- **未决**：即梦的接入方式 —— 走**火山引擎官方 API** 还是仅有网页端？若仅网页端，云接入方案需换思路
  （此项待确认后影响 M6）。
