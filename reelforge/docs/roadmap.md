# 里程碑与未决问题

## MVP 垂直切片

主链路：
> 描述角色 → 出多视角定稿 → 锁成角色档案 → 给一个场景生成 2~3 个分镜关键帧 →
> 对选中关键帧图生视频出几条 take → 选片 → **导出工程 / 片段**

只要这条跑通，就证明了「Agent 管一致性 + 管繁琐参数 + 多反馈」的核心价值；音轨/时间线/口型/更多配方
都是后续加法。

## 里程碑

- **M1 — Orchestrator 骨架**：FastAPI + Claude tool-use + 接现有 ComfyUI；图编辑原语 +
  `validate`（对照 `/object_info`）+ `run`；跑通「自然语言 → 选配方 → 校验 → 出一张人物定稿图」。
  *不依赖 SwiftUI 画布，最快验证「Agent 搭图」是否靠谱。*
- **M2 — 一致性**：角色档案（定稿图集 + identity + 触发词，必要时 LoRA）+ 自动注入；多视角定稿配方。
- **M3 — 分镜 + 图生视频**：关键帧合成配方 + 图生视频配方（本地 Wan）+ 多 take。
- **M4 — SwiftUI 客户端**：对话侧栏 + 双层画布（展示 + 轻编辑）+ 可切换布局 + 远程连接配置。
- **M5 — 反馈链路**：远程 WS → 节点进度 / 中间预览 / 产物时间线 / 重跑 / 选片对比。
- **M6 — 云路由**：接入即梦或 Qwen，打通本地 ↔ 云路由 + 费用预估。
- **(后续) M7+ — 剪辑 Agent**：剪辑 / 音轨 / 口型同步 / 调色 / 字幕 / 和片。

## 风险

- **可靠性**：自动搭图出错率 —— 靠「配方 + schema 校验」压住。
- **一致性质量**：跨镜头角色 / 服装 / 风格漂移 —— M2 重点验证。
- **SwiftUI 自研画布工作量**：分阶段铺能力，MVP 只做展示 + 轻编辑。
- **云成本与限流**：必须预估 + 确认 + 配额保护。
