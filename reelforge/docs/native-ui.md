## 待讨论的核心问题（候选清单）

下面是我认为原生 UI 真正要解决的几个核心矛盾，供下一轮讨论时取舍：

1. **双外壳如何统一**：对话优先（入门）与画布优先（生产力）共享同一数据模型，布局如何无缝切换、
   各自默认露出多少？怎样让同一个用户在「省心」和「掌控」之间平滑过渡？

2. **双层无限画布的导航**：项目层（角色/资产/镜头关系）⇄ 工作流层（ComfyUI 节点图）如何缩放/钻取/
   返回而不迷路？面包屑、缩略地图（minimap）、聚焦模式怎么设计？

3. **自研无限画布的性能与交互**：SwiftUI + Metal 下平移/缩放/大量节点/连线的渲染与命中测试；
   节点视图与连线层如何分工；轻编辑（改 widget、连/断线、增删节点）的手势与可达性。

4. **Agent 与画布的协同呈现**：Agent 的「思考 / 工具调用 / 改图 / 提问」如何在画布上可视化（高亮被改的
   节点、diff 预览、可接受/撤销）？人接管与 Agent 接管如何切换且不冲突？

5. **「多反馈」的信息架构**：进度、中间预览、产物时间线、选片对比、费用——这么多反馈如何在不淹没用户的
   前提下随时可见？哪些贴在节点上、哪些进侧栏、哪些进独立面板？

6. **一致性资产的呈现**：Character Bible / 资产库在 UI 里怎么放，才能让「自动注入一致性」既透明又可控？

7. **入门引导 vs 专家效率**：渐进式披露、模板入口、快捷键/批量操作如何兼顾两类用户？

8. **远程感知**：client/server 形态下，连接状态、传输进度、服务器忙闲、断线恢复如何在 UI 里如实呈现？

## 设计原则（初步，待确认）

- 画布是「事实来源」的可视层，不是另一份状态。
- 反馈优先：用户任何时候都能看到「在干什么、到哪了、长出了什么」。
- Agent 可解释、可撤销；人随时能接管。
