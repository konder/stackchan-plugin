# 架构 / 技术栈 / 工程要点

## 1. 总体架构（client / server）

RTX 5090 在**本地千兆局域网的独立 Linux 机器**上，因此是「Mac 客户端 + 5090 渲染/编排服务器」。
Mac 只负责 UI、画布与预览；重活（推理、编排、Agent、素材存储）都在 5090 机器上。

```
   ┌──────────────── Mac (SwiftUI 客户端) ────────────────┐
   │  可切换布局: 对话优先 ⇄ 画布优先 (同一数据模型)        │
   │  ┌───────────┐   ┌──────────────────────────────┐   │
   │  │ Chat/Agent│   │ 自研无限画布 (SwiftUI+Metal) │   │
   │  │ 侧栏      │   │ 项目层 ⇄ 工作流层 双层视图     │   │
   │  └─────┬─────┘   └───────────────┬──────────────┘   │
   │        └──────── Project / Graph IR (唯一事实来源) ──┘ │
   └───────────────────────┬──────────────────────────────┘
                            │  HTTP/SSE/WS (千兆局域网直连)
   ┌────────────── 5090 机器 (Linux, headless) ────────────┐
   │  ┌──────────────────────────┐                         │
   │  │ Orchestrator (Python/     │  ← Agent 运行时在这     │
   │  │ FastAPI)                  │                         │
   │  │ - 导演 Agent + 搭图 Agent │                         │
   │  │ - 配方库 + /object_info   │                         │
   │  │   schema 索引(RAG) + 校验 │                         │
   │  │ - 任务队列 / 成本估算 / 路由│                        │
   │  │ - 项目存储 / 资产库        │                         │
   │  └───────┬───────────┬──────┘                         │
   │   ┌──────▼─────┐  ┌──▼──────────────────────────┐    │
   │   │ ComfyUI    │  │ 云适配器                      │    │
   │   │ (本地5090) │  │ 即梦(火山引擎) / Qwen(DashScope)│  │
   │   └────────────┘  └─────────────────────────────┘    │
   └───────────────────────────────────────────────────────┘
```

**为什么 Agent / 编排放在 5090 机器上**：改图前要对照 ComfyUI 的 `/object_info` 真实 schema 做校验、
要跑 RAG 检索配方、要管云任务轮询、要读写大体积素材——这些在 Python 生态里最省事，且离 ComfyUI 和
资产最近。Mac 端保持纯客户端，将来加 iPad / Web 客户端不用重写后端。

## 2. 技术栈

- **Mac 客户端**：SwiftUI（自研无限画布）+ Metal（连线 / 大图层渲染）+ AVKit（视频预览）。
  云密钥与服务器凭据存 **Keychain**。
- **Orchestrator**：Python + FastAPI，和 ComfyUI 同机（5090 Linux）。承载 Claude tool-use 循环、
  配方库、schema 索引（RAG）、任务队列、云适配器、项目 / 资产存储。
- **Agent 模型**：Claude（Fable 5 / Opus）+ tool use。
- **执行引擎**：ComfyUI（本地 5090，Linux headless）。
- **云**：即梦（火山引擎 API）、Qwen 通义万相（DashScope / 百炼）；异步作业模型
  `submit → task_id → poll → 下载`。

## 3. client / server 工程要点

- **连接**：本地千兆局域网直连 Mac ↔ 5090；若需外网访问再加 Tailscale 之类隧道，避免裸公网暴露
  ComfyUI 与 Orchestrator。
- **资产传输**：Orchestrator/ComfyUI 机器是产物的存储 + 下载源，Mac 端做本地缓存；千兆 LAN 下大视频
  传输不是瓶颈。约定缩略图/低码率预览先行、原始大文件按需拉取。
- **断线恢复**：任务在服务器跑且状态持久化，Mac 重连后能拉回进行中任务的状态与已完成产物。
- **5090 环境**：Blackwell 需较新 CUDA / PyTorch；Linux headless 跑 ComfyUI。
- **多客户端友好**：后端无状态化交互 + 服务器侧持久化，为将来 iPad/Web 客户端留路。

## 4. 许可 / 分发

- 站在 **ComfyUI（GPL-3.0）** 上做执行引擎。SwiftUI 客户端经 HTTP/WS 通信、属独立程序，
  不构成衍生作品，**客户端可自选许可**（MIT 等）；若编写 ComfyUI 自定义节点，该部分受 GPL 约束。
- 自有渠道分发（不上 App Store）→ 无沙盒限制；需做 **Apple 公证 / 签名**，否则 Gatekeeper 拦截。
- 云密钥默认存 Keychain，不明文落盘。
